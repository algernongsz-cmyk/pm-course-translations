简介

Locker 是一个自托管的文件存储平台替代方案，可以替代 Dropbox 和 Google Drive。您可以从自己的基础设施上传、组织和共享文件。

支持自带存储 - 本地磁盘、S3、R2 或 Vercel Blob。只需一个环境变量即可切换。通过 tRPC 提供完整的类型安全 API。

核心特性

🔓 存储提供商无关

通过单个环境变量在本地文件系统、AWS S3、Cloudflare R2 或 Vercel Blob 之间切换。您的数据，您的基础设施。

🔍 图片和 PDF 内容搜索

根据文件内容而不仅仅是名称查找文件。Locker 将图片和 PDF 转录为可搜索文本，让每个文档都可被发现。

💻 虚拟 Bash Shell

使用熟悉的命令导航您的文件。通过虚拟文件系统 API 使用 ls、cd、find、cat 和 grep。

👥 工作空间团队

邀请团队成员并设置基于角色的访问权限。跨工作空间组织文件，具有细粒度的权限控制。

🔐 默认安全

支持邮箱/密码和 Google OAuth 身份验证。会话在服务器端管理，使用加密 cookie。

🔑 API 密钥

通过 API 密钥以编程方式访问您的文件。构建集成并自动化工作流程，具有完整的 tRPC 类型安全。

为什么要自托管？

完全控制您的数据 - 没有供应商锁定，没有意外定价，没有第三方读取您的文件。在您自己的服务器上部署，将一切掌握在自己手中。

常见问题

什么是 Locker？

Locker 是一个开源的、可自托管的文件存储平台。可以把它看作是您完全控制的 Dropbox 或 Google Drive - 在您自己的服务器上部署，使用您自己的存储后端，拥有您的数据。

支持哪些存储提供商？

Locker 开箱即支持本地文件系统、AWS S3、Cloudflare R2 和 Vercel Blob。您可以通过更改单个环境变量在它们之间切换。无需修改代码。

Locker 免费吗？

是的。Locker 是免费且开源的。您可以免费用于个人项目、团队或整个组织。您只需支付选择托管的基础设施费用。

分享链接如何工作？

您可以为任何文件或文件夹生成可分享链接。每个链接可以选择设置密码保护、过期日期和最大下载次数。接收者无需账户即可访问共享文件。

其他人可以上传文件给我吗？

可以。上传链接允许任何人在无需创建账户的情况下将文件发送到您的存储。非常适合从客户、学生或协作者收集文档。

技术栈是什么？

Next.js 16 使用 App Router、PostgreSQL 配合 Drizzle ORM、tRPC 用于类型安全 API、BetterAuth 用于身份验证、Tailwind CSS 用于 UI。项目使用 Turborepo monorepo 配合 pnpm workspaces。

如何部署 Locker？

克隆仓库，运行 pnpm install，启动 PostgreSQL 数据库，配置 .env 文件，运行迁移，然后启动服务器。默认配置适合本地开发。对于生产环境，可以部署到任何兼容 Node.js 的平台。

准备好拥有您的文件了吗？

几分钟内部署 Locker，重新掌控您的文件存储。永久免费、开源、无任何附加条件。
