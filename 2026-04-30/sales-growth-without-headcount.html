AI 是解决销售增长但不增加人力问题的关键答案

销售团队被要求提供更多 pipeline，无论人力是否跟进。AI Agents 是对这个限制的第一个真正答案。

我经历过足够多的规划周期，知道销售领导者所面临的不可能完成的数学题。

每年，我们都被要求提供更多的 pipeline，期望团队能找到实现目标的方法，无论人力是否跟进。

在好年景里，你可以缩小一些差距，但潜在的限制保持不变：你的 pipeline 上限与你的员工数量挂钩。要求越来越高，但满足要求的资源却没有同步增长。对于"如何在不增加 30% 人力的情况下增长 30% 的 pipeline"这个问题，从来就没有真正的答案。

在我 20 年的销售生涯中，这个问题第一次有了答案，而这个答案来自于我们如何使用我们的客户 Agent Fin 来进行入站销售。

一个新的 pipeline 来源

关于销售中 AI 的讨论往往集中在更快地完成同样的工作，但这忽略了拥有 Agent 的真正价值所在。我们在实践中看到的是，Agent 正在产生自己的 pipeline。

很容易将其视为 SDR 团队内部的另一个效率层。然而，AI 生成的 pipeline 应该被视为一个独立的来源，与已经产生的内容并列。它需要被单独跟踪，有自己目标和负责人。它是对人类所产生内容的补充。

在 Intercom，我们的运作方式如下。Fin 有自己的绩效指标，但它被要求达到与团队中任何代表相同的目标。在实时聊天中，我们要看有多少对话被合格、不合格和流失。但结果与传统销售动作相同：预约会议、创建 pipeline、产生收入。

如果你将其折叠到现有团队的数字中，你就无法看清 Agent 实际在做什么。这种重新构建改变了你与高管团队和董事会的对话方式。推销不再关于效率提升，而是关于一个以前不存在的新 pipeline 来源。

上个月是我们从 Fin 获得的 pipeline 最高的月份，比仅由人类处理实时聊天时更好。

客户服务的类比

在我们为销售实施 AI 之前，我与客户支持团队密切合作。他们制定了我们今天正在遵循的 AI 转型路线图。

使其成功的是围绕它的组织架构。有专门的 AI 动作负责人，客户支持代表和 Agents 并行运行。有人对 Agent 的表现负责。他们没有进行一次性设置，而是实施了持续优化循环来提高其性能。这些经验直接转移到销售中。

"正确的基准是在该渠道上持续、大规模地匹配高绩效代表"

流程比人们预期的更类似于销售。在这两个职能中，你都在确认需求并努力寻找正确的解决方案。目标不同，但工作的基本形状重叠。

我从在客户服务中实施 Agent 所理解的是，正确的基准是在该渠道上持续、大规模地匹配高绩效代表。当 Agent 做到这一点时，收益会复合增长，它释放出时间来做关系真正重要的工作。

SDR 的问题

这是我最常被问到的问题。如果 Agent 负责前线，SDR 实际上在做什么？

Agent 处理前线入站。它立即参与、合格、将高意向潜在客户路由到正确的团队，并通过将低意向访客引导到自助资源或记住他们的上下文直到他们准备好进行真正的对话来保持他们的热度。它在任何时间、跨语言、没有人类团队带来的容量限制的情况下做到这一点。

"这是他们现在很少有容量做的工作，因为他们的时间太多花在前线上。Fin 改变了这一点"

改变的是 SDR 时间去向。对我们来说，那是基于电话的合格。当他们与潜在客户通话时，这是我们仍然看到最强转化的地方。它还涉及在账户中跨多个利益相关者建立关系，这种多线程参与需要时间和判断。角色开始看起来更像顾问。

试用是一个很好的例子。SDR 可以帮助潜在客户从中获得真正价值，而不是将试用视为转化机制。这是他们现在很少有容量做的工作，因为他们的时间太多花在前线上。Fin 改变了这一点。

我想直接说一件事。一些公司正在转向完全用 AI 取代他们的 SDR 职能。我认为那是个错误。SDR 是关闭团队的人才管道。成为你最好的 AE 的代表，更多时候是那些通过 SDR 角色成长起来的人。那是他们学会快速合格和建立关系的地方。为了降低成本而取消该职能会对 funnel 上游产生影响，这些影响需要几年才会显现。

机会不仅仅是效率，而是一个能自己生成 pipeline、大规模参与潜在客户、并给 SDR 容量发展对他们职业生涯下一步重要的技能的 Sales Agent。

当前状况

许多销售组织仍处于早期阶段。初创公司和小公司处于领先地位。他们正在从头开始构建 AI-first，有意设计不需要以传统方式扩展人力。

更大、更成熟的销售开发职能主要仍在运行标准工作流。这有道理，因为转型一个成熟组织比从头开始构建一个新组织更复杂。这种复杂性不是等待的理由。势头已经在建立，开始显示出正在拥抱 AI 的团队和不拥抱的团队之间的差距。

"领先的组织让一个人负责 AI 动作并让他对其负责"

我开始看到销售团队内部出现专门的 AI 负责人。这需要一个对 Agent 实际表现负责的项目级人员，而不是将 AI 工具附加到现有工作描述上。在 Intercom，我们创建了这个角色——它被称为"AI SDR program lead"。这个角色拥有 Fin 在入站 SDR 动作中的策略、实施和优化，确保它推动 pipeline 增长并在我们的系统和工作流中良好整合。这是直接来自 AI 动作的新职业机会，我们现有的一位经理调入了这个职位。

这反映了客户服务转型刚开始时发生的事情。领先的组织让一个人负责 AI 动作并让他对其负责。

限制已经改变

假设 pipeline 增长需要按比例增加人力，这在我从业以来的销售界一直被视为固定法则，但情况已不再如此。

AI 生成的 pipeline 是真实且可衡量的，并且它随着投资和负责而改善，就像职能的任何其他部分一样。将其视为独立的来源，有自己的目标和专门的负责人，是微小的效率提升与真正打破 pipeline 增长与人力之间的联系之间的区别。

限制不会消失，但会转移。它不再仅仅关于你能增加多少人。它关于 Agent 对你的产品、你的客户以及你如何合格需求的理解程度如何。

第一次，pipeline 上限可以高于你的人力允许的水平。
