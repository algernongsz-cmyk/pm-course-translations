Agent 已经能够胜任工作了

如今的 Agent 比以往任何时候都更具能力。当企业在部署它们时遇到困难，这反映的是组织准备度的问题。

Jennifer Murphy
⏱ 阅读时间：5分钟

在每一次 AI Agent 评估的背后，都隐藏着一个问题：它能做什么？

两年前，这是一个正确的问题，因为当时的 Agent 能力有限，能力确实是一个真正的约束条件。组织需求与技术能力之间的差距很大。

这个差距此后已经大大缩小，然而大多数组织在运行其 Agent 时，远低于技术上的可能性。他们只是用它们来回答和路由，但不再进一步——不去查找信息、采取行动，或解决复杂的多步骤问题。

对此的标准解释是 AI 还不够好，无论是模型需要改进还是供应商需要提供更多功能。但通过研究各行各业积极扩展其 AI 自动化的组织后，我们发现这种解释并不像人们认为的那样普遍成立。

我们观察到的团队主要不是受限于其 AI 能做什么，而是受限于其组织结构允许它做什么。

五种准备度

针对 AI 的"准备度"可以分为五种不同的类型，大多数组织具备部分但不是全部。

  1. 内容准备度是指你能否清晰、一致地解释你的产品和政策。大多数公司都能做到。
  2. 范围准备度是指你是否定义了边界：AI 应该何时介入，何时应该退让？边缘情况会倍增，意图因客户群体而异，敏感话题会在对话中浮现，但大多数团队通过努力可以解决这个问题。
  3. 流程准备度是事情开始变得困难的地方。这是指你能否清晰地表述你的流程，让一个没有多年隐性知识的人能够遵循。快乐路径很少是问题。问题在于失败路径、决策分支、从未被记录的变体，因为它们一直存在于某人的脑海中。
  4. 数据准备度是第一个真正的悬崖。你能否在需要做出决策时可靠地识别正确的用户、账户或对象？数据在实时情况下是否可信？API 是否稳定、可访问并真正连接？对大多数组织来说，诚实的回答是"部分可以，但我们并不总是确定它何时会出问题。"
  5. 执行准备度是最高的门槛。不仅仅是技术层面（Agent 能否做出更改？），更是组织层面。当错误的退款被处理时谁负责？谁来检测？谁来恢复？是否有拥有权威的人真正接受这个风险？

大多数公司具备前两项，有些具备第三项，更少具备第四和第五项。

当团队试图深入时会发生什么

我们研究了六个行业的公司——能源、医疗、电商、游戏、金融服务、物业管理——都在尝试扩展其 Agent 的能力。

模式是：团队着手自动化真实行动——查询账户状态、处理变更、处理交易。在大多数情况下，AI 在技术上可以做到，但在某个节点（介于引导用户完成流程和代表用户查找信息之间的某处），他们碰壁了。

一个团队尝试自动化应用变更，但无法在其内部系统中可靠地识别要修改哪个应用。另一个团队探索账单自动化，但由于监管约束无法访问实时账户数据。第三个团队需要验证其 Agent 无法可靠触达的第三方供应商系统中的状态。

在大多数情况下，团队根据其基础设施支持的范围重新设计。他们转向引导——逐步引导用户完成流程，而不是代表他们执行变更。这有效，它解决了对话并交付了真实价值，只是与任何人计划的方式不同。

评估陷阱

大多数 Agent 评估都围绕能力构建。它能处理复杂查询吗？它支持多渠道吗？它能与我们的系统集成吗？

这些是合理的评估项，但它们产生的是能力评分，而这并不能告诉你你的组织是否能真正使用你购买的东西。

那些达到更深层次自动化的团队，那些早期就开始执行行动的团队，并没有"更好的 AI"，他们拥有更标准化的运营。已经定义清晰、持续应用、并通过稳定系统暴露且有明确规则的操作。自动化不是在发明新行为，它只是在触发已经在其他地方被严格控制的行为。

准备度决定能力，而不是反过来。这将评估问题从"AI 能做这个吗？"重新框定为"我们真的准备好让它做了吗？"

比你想象的更准备好

在大多数关于 AI 准备度的对话中，有一个常被忽视的事实：组织通常比他们假设的更进一步，只是不是为他们计划的那种工作。

一个团队着手自动化退款，但能够可靠地引导用户完成复杂的故障排查，这已经部署了真正的能力。他们在其准备度支持的层面上运营，这是一个起点，而不是缺陷。

更有用的框架不是"我们准备好了吗？"——而是"我们准备好了什么，以及具体是什么阻碍了从这里到下一级？"差距往往是具体的：缺失的 API、存在于三个不一致系统中的数据、从未被记录的流程，或没人回答的所有权问题。这些是可解决的问题。它们需要的投资方式与购买更有能力的 Agent 不同。

悬而未决的问题

目前还没有人认真思考过组织实际上如何构建准备度。

它是通过先在较浅层次使用 AI 自然发展出来的吗？还是主要取决于先前的决策，比如几年前做出的系统架构选择、随时间积累的运营成熟度、与 AI 无关的工程投资？当准备度确实增加时，实际上发生了什么变化？是支持团队发展出来的吗？是工程团队授予的吗？是否需要高管赞助和对没有明显 AI 标签的基础设施投资？

在有更清晰的答案之前，这个模式很可能会继续。公司会购买有能力的 Agent，计划雄心勃勃的推出，然后发现更难的工作是构建组织基础设施。

Agent 可以胜任工作。问题在于让它们胜任需要付出什么。
