Mythos, Muse 与计算的机会成本

原文：Mythos, Muse, and the Opportunity Cost of Compute

作者：Ben Thompson | 发布日期：2026年4月13日

翻译日期：2026年4月23日

2025年1月，Doug O'Laughlin 在 Fabricated Knowledge 上宣称，o1 和推理模型标志着 Aggregation Theory（聚合理论）的终结：

我认为，除了经济因素外，模型的改进没有实际限制，而这将是未来真正的约束。如果我们在一个模型上投入无限的资金，它就会得到改进，这是合理的。问题在于，无限的资金对一个企业来说是否有意义。

这将是2025年的关键问题。AI 的经济学如何使其可行？互联网的一个核心假设刚刚被打破。边际成本再次存在，这意味着大多数超大规模服务商将变得越来越资本密集。

Aggregation Theory 的时代已经过去，AI 再次让技术变得昂贵。消费增加导致的成本增加这种关系是反互联网时代的思维。这将是要面对的重大问题。超大规模服务商的商业模式主要建立在边际成本为零的基础上。因此，只要建立基础设施，并在一个互联网规模的产品中填充用户，你就可以赚钱。

这个时代即将结束，未来将更加奇怪、更加计算密集。回顾2010年代，我们可能会认为那是技术长河中一个天真的时代。我们关于这一时期的一个基本假设正在瓦解。这将是未来技术领域最重要的变化。

Aggregation Theory，如果我可以这么说的话，是理解2010年代的最佳方式，尤其是消费科技。它解释了支撑 Google 和 Facebook 主导地位的动态，以及 App Store 和 Amazon 电商业务；它也是一个有用的（虽然不完整）框架，用于理解一整套消费服务，如 Uber、Airbnb 和 Netflix。

然而，值得指出的是，支撑 Aggregation Theory 的一些关键见解要古老得多，并且根植于科技本身的根本性质。正如 O'Laughlin 所指出的，它们根植于零边际成本的概念。

边际成本

边际成本是生产一个额外单位商品的成本。考虑一个制造小部件的工厂：

  • 你需要土地来建工厂
  • 你需要机器来装备工厂
  • 你需要电力来运转机器
  • 你需要人来操作机器
  • 你需要原材料来制造小部件

土地和机器显然是固定成本；你必须拥有这两者才能开始生产，无论你是否多生产一个小部件，你都在为这两者付费。另一方面，原材料显然是边际成本：如果你多生产一个小部件，你就需要多一个小部件的原材料。当涉及到实物商品时，电力和人力也是边际成本：你需要更多或更少的电力和人力，取决于你生产更多还是更少的小部件。

边际成本的重要性在于它们提供了一个价格下限。公司会亏损经营，因为利润和亏损是一个包含折旧的会计概念，即你的固定成本。例如，想象一家公司花费1000美元建了一个工厂，生产边际成本为10美元的小部件：只要小部件的价格>10美元，公司就会继续生产，即使他们赚的钱不足以覆盖折旧成本（即他们亏损经营），因为至少他们在每个小部件上仍然获得了边际利润（公司可能不会做的是投资任何更多的固定成本，并且最终可能会因为可能为这些固定成本融资的债务利息而破产）。

我解释这一切恰恰因为它与科技几乎完全无关。首先，通常没有原材料成本，因为输出是数字化的。其次，因为没有原材料成本，而且固定成本非常大，电力和人力通常被视为固定成本，而不是边际成本：当然你会全天候满负荷运行你的服务器，因为你能产生的每一分额外收入都是值得的。

AI 非常符合这一范式：输出是数字化的，虽然 AI 芯片消耗大量电力，但这只是芯片本身成本的一小部分，也就是说，拥有 AI 芯片的人不会在利用它们方面做边际成本计算。它们会被使用！相反，重要的决定是它们将被用于什么。

机会成本

考虑 Microsoft：上个季度，该公司未达到华尔街的 Azure 增长预期，不是因为需求不足，而是因为公司决定将其产能用于自己的产品。CFO Amy Hood 在公司的财报电话会议上 表示：

我认为最好将我们给出的 Azure 指导视为一个分配产能指南，关于我们可以在 Azure 收入中交付什么。因为当我们花费资本，特别是投放 GPU 时，这也适用于 CPU，特别是 GPU，我们正在做出长期决策。我们要做的第一件事是解决销售中使用量的增加以及 M365 Copilot 和 GitHub Copilot 加速步伐的问题，这是我们的第一方应用程序。然后我们要确保我们在 R&D 和产品创新的长期性质上进行投资。我认为你从我们这里看到的加速，以及过去一段时间的许多产品，都是因为我们将 GPU 和产能分配给了我们在过去几年中雇佣的许多有才华的 AI 人员。

然后，当你结束时，结果是，你最终将剩余部分用于服务 Azure 产能，这在需求方面继续增长。一种思考方式是，因为我认为，我有时会被问到这个问题，如果我将在 Q1 和 Q2 刚刚上线的 GPU 全部分配给 Azure，KPI 就会超过40。我认为最重要的是意识到这是关于投资使客户受益的堆栈的所有层。我认为这对于思考资本增长是有帮助的，它体现在每一块中，它体现在整个业务的收入增长中，也体现在 OpEx 增长中，因为我们在投资我们的人员。

Microsoft 在这里面临的成本不是边际成本，而是机会成本：在一个领域使用的计算不能在另一个领域使用；就这些收益而言，Microsoft 承认，如果他们愿意，他们本可以达到 Azure 的数字，但选择优先考虑自己的工作负载，因为正如 CEO Satya Nadella 后来在电话会议中指出的那样，这些工作负载具有更高的毛利率和更高的生命周期价值。

超大规模服务商面临的挑战是机会成本，而不是边际成本。多少计算应该给客户，给哪些客户？多少应该保留给内部工作负载？Microsoft 需要平衡 Azure——既为其企业客户也为 OpenAI——以及其软件业务；Amazon 需要平衡其电商业务、AWS 以及其在 Anthropic 和 OpenAI 的战略投资。Google 需要平衡 GCP、其自己在 Anthropic 的战略投资以及其消费业务。

Mythos

上周，Anthropic 发布了 Mythos，这是其最先进的模型。而且，以典型的 Anthropic 风格，它是通过关注其危险性来宣布的；从 Project Glasswing 的介绍性文章开始，这是该公司利用 Mythos 解决安全问题的一项计划：

我们组建 Project Glasswing 是因为我们在 Anthropic 训练的一个新前沿模型中观察到了一些能力，我们相信这些能力可能会重塑网络安全。Claude Mythos Preview 是一个通用的、未发布的前沿模型，它揭示了一个严峻的事实：AI 模型已经达到了这样的编程能力水平，以至于它们可以在发现和利用软件漏洞方面超过几乎所有最熟练的人类。

Mythos Preview 已经发现了数千个高严重性漏洞，包括每个主要操作系统和网络浏览器中的一些。考虑到 AI 进步的速度，这种能力扩散到那些致力于安全部署它们的参与者之外，将不会很久。其后果——对经济、公共安全和国家安全——可能是严重的。Project Glasswing 是一个紧急尝试，旨在将这些能力用于防御目的。

在 上周的一篇更新中，我将 Anthropic 的"灾难色情作为营销工具"的方法类比于《狼来了》；这个类比的重要之处不仅在于男孩发出了虚假警报，而且最终狼确实来了。为此，我在两周前写道，支撑所有软件的无数安全问题，以及我对 AI 将在长期内解决这些问题的乐观态度，即使它在短期内会让事情变得更糟。换句话说，Mythos 是否代表重大安全威胁其实并不重要：如果这个模型不代表，未来的模型会代表；为此，我确实支持利用 Mythos 在不良行为者发现和利用漏洞之前主动发现和修复漏洞。

同时，也值得注意的是，Anthropic 不广泛提供 Mythos 还有其他原因，将访问限制在有限数量的具有高能力和支付意愿的公司。首先是那些机会成本：Anthropic 已经在服务其当前模型方面缺乏计算；X 上充斥着关于 Anthropic 据称在过去一个月左右让 Claude 变笨 的投诉和争论。让 Mythos 更广泛地可用——特别是对不按使用付费的订阅计划——会让情况变得更糟。

换句话说，Anthropic 面临的不是边际成本问题，而是机会成本问题：将其计算分配到哪里。当然这可能变成利润率问题：我怀疑 Anthropic 将通过从超大规模服务商和新云获取更多计算来克服 其在计算方面的保守主义，并为此付出高昂的代价。

处理这些成本的关键将是未来对 Claude 收取更高的费用；这反过来意味着维持定价权，这导致了不广泛发布 Mythos 的第二个好处。Anthropic 当然面临来自 OpenAI 的竞争；然而，对于这两个前沿实验室来说，长期来看真正的竞争是开源模型。目前这些主要来自中国，而快速跟进前沿模型的一个关键要素是蒸馏；来自 Anthropic 的博客：

我们已经确定了来自三个 AI 实验室——DeepSeek、Moonshot 和 MiniMax——的大规模非法提取 Claude 能力以改进其自身模型的活动。这些实验室通过约24,000个欺诈账户生成了超过1600万次与 Claude 的交互，违反了我们的服务条款和区域访问限制。

这些实验室使用了一种称为"蒸馏"的技术，这涉及在更强模型的输出上训练一个能力较弱的模型。蒸馏是一种广泛使用且合法的训练方法。例如，前沿 AI 实验室经常蒸馏他们自己的模型，为客户创建更小、更便宜的版本。但蒸馏也可能被用于非法目的：竞争对手可以利用它在比独立开发所需的时间和成本少得多的情况下，从其他实验室获得强大的能力。

我绝对相信这是一个真正的问题，并且 在去年发布 DeepSeek R1 时就写道了这一点。我也认为，除了前沿实验室之外的每个人的利益都是假装这不是真的；开源模型不受前沿实验室的加价或计算限制，这正是大多数公司有可用的开源模型的原因，无论它们是否经过蒸馏。当然，这并不意味着它们运行是免费的：你仍然需要提供计算。

然而，请注意，这使阻止蒸馏成为前沿实验室更重要的优先事项：首先，他们想要保护他们的利润率。其次，然而，他们最大的成本是机会成本：他们无法服务的客户，因为他们没有足够的计算。在某种程度上，他们可以让计算对其潜在客户来说变得不那么有用——通过阻止开源模型蒸馏他们的模型——就是在一定程度上他们可以以更优惠的费率为自己获取该计算。

Meta Muse

Mythos 并不是上周宣布的唯一新模型：Meta 也发布了他们新前沿实验室的第一个成果。来自 该公司博客文章：

今天，我们很高兴介绍 Muse Spark，这是由 Meta Superintelligence Labs 开发的 Muse 模型家族中的第一个。Muse Spark 是一个原生多模态推理模型，支持工具使用、视觉思维链和多智能体编排。

Muse Spark 是我们扩展阶梯上的第一步，也是我们 AI 工作全面改革的第一款产品。为了支持进一步的扩展，我们正在整个堆栈中进行战略投资——从研究和模型训练到基础设施，包括 Hyperion 数据中心……

Muse Spark 在多模态感知、推理、健康和智能体任务方面提供了具有竞争力的性能。我们继续在当前存在性能差距的领域进行投资，例如长期视野的智能体系统和编码工作流程。

Muse Spark 不是最先进的，但它进入了竞技场，总体上是 Meta Superintelligence Labs 的一个积极的第一印象。然而，对我来说最值得注意的是，过去九个月的 AI 已经清楚地表明，CEO Mark Zuckerberg 决定开始"[Meta 的] AI 工作的全面改革"是正确的。

O'Laughlin 以此开头的文章的触发因素是推理，使用更多 token 的模型会产生更好的答案；从那时起，智能体指数级增加了 token 需求，因为它们可以在没有人类参与的情况下持续使用 LLM。这是 Claude 需求飙升的巨大推动力，以及 OpenAI 的 Codex。此外，这个用例是如此潜在有利可图，以至于不仅 Anthropic 的收入飙升，而且 OpenAI 正在将其重点转向企业。

事实上，你可以认为 OpenAI 最大的挑战之一是它在 ChatGPT 中拥有如此受欢迎的消费产品。我用我的 Aggregation Theory 视角长期坚持认为，那个用户群是 OpenAI 的巨大优势，但这假设公司可以有效地将其货币化，这也是为什么 我一直如此强烈地主张广告模式。OpenAI 对此有很大的预测，但在那实现之前，那个大型消费群在 OpenAI 的重点和计算方面是一个巨大的机会成本。该公司已经——在其信用下，并面对广泛的质疑——在更多计算方面进行了重大投资，但将越来越多的计算分配给企业愿意付费的智能体用例，即使以牺牲消费业务为代价的诱惑也将非常大。

这让 Meta 处于相对于行业其他人的独特位置：与任何超大规模服务商或前沿实验室不同，Meta 没有企业或云业务需要担心。这意味着服务消费市场不会带来机会成本。当然，这些机会成本本来也会小得多，因为 Meta 已经有一个规模化广告业务来货币化使用。换句话说，Meta 实际上可能在赢得消费空间方面面临比几个月前看起来更少的竞争，仅仅因为那是他们的主要关注点——而且因为他们有自己的模型，这意味着他们不需要担心无法访问前沿实验室（这种分析的许多内容当然也适用于 Google）。

这同样也是为什么 Meta 应该开源 Muse，就像他们开源 Llama 一样。最因前沿模型的广泛可用性而受到损害的实体是其他前沿实验室，他们将看到定价权降低，并面临对计算的竞争加剧。这将使他们更难承受追求消费市场的机会成本，将其留给 Meta。

需求与供给

那么"Aggregation Theory 的时代……已经过去"了吗？一方面，认为创造和维护价值的方式将来自拥有客户的见解几乎肯定会继续存在。在消费方面，拥有客户会导致广告，这提供收入来为客户提供服务。在企业方面——我要指出，这从来不是 Aggregation Theory 旨在应用的领域——我认为 Anthropic 和 OpenAI 都可能继续向上移动堆栈，提供与软件供应商直接竞争的功能（这种方法也符合不公开提供领先模型）。

另一方面，O'Laughlin 关于我们正在并将继续受到计算限制的观察是一个重要的观察：公司将无法假设他们可以服务所有人，因为服务一组客户会带来不服务另一组的机会成本。这不会——至少在理论上——永远持续下去：在某个时候，AI 将对足够多的用例"足够好"，以至于将有足够的计算能力来利用服务于 AI 并不涉及重大边际成本这一事实；然而，那个理论上的未来感觉比以往任何时候都更远。

OpenAI 正在押注，这种计算限制——以及他们为克服它所做的交易——将比 Anthropic 目前在终端用户中的势头更重要。来自 Bloomberg：

OpenAI 本周告诉投资者，其早期大力增加计算资源的努力使其在 Anthropic PBC 面前获得了关键优势，而此时其长期竞争对手正在获得进展并考虑潜在上市。

这家 ChatGPT 制造商表示，它通过"快速且持续地"增加计算能力来支持其软件的更广泛采用，从而超越了 Anthropic，根据该公司在 Anthropic 宣布更强大的 AI 模型 Mythos 后发送给一些投资者的一份备忘录。这份雄心勃勃的基础设施建设——被一些人批评为过于昂贵——使 OpenAI 能够更好地跟上对 AI 产品不断增长的需求，备忘录指出。

我不太确定这会是决定性的。当涉及到 AI 时，分发和交易成本仍然是免费的——这是聚合者的两个前提条件——这意味着赢家应该是那些拥有最引人注目产品的公司。那些产品将赢得最多的用户，提供获得服务他们的计算所需的资金；考虑 Anthropic 确保获得 TPU 供应的重要部分的交易，鉴于 TSMC 的产能限制，这最终是从 Google 那里获取供给的例子。我怀疑 Anthropic 可以获取更多，包括已经建成的超大规模服务商和新云产能。是的，那些计算会更昂贵，但如果需求足够高，必要的现金流将会存在。

换句话说，我赌的是拥有需求最终将胜过拥有供给，这表明 Aggregation Theory 的基本原则继续存在。换句话说，我认为 OpenAI 将需要通过更好的产品获胜，而不仅仅是更多的计算；话又说回来，如果更多计算是更好产品的关键，那么供给最重要吗？无论如何，他们肯定会专注于向正在推动 Anthropic 惊人增长的企业客户交付这两者。真正的成本可能是他们目前主导的消费市场，因为 Meta 没有什么可失去的，一切都可赢得。
