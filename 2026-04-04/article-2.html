推出 Fin API 平台

来源：Intercom Blog 原文：链接

今天我们宣布推出 Fin API 平台。

我们一流的垂直客户服务模型（这些模型为 Fin 提供动力）现在可供您混合搭配，并以疯狂的规模部署，为您创建完美的客户 Agent。在这篇文章中，我将准确解释我们宣布的内容、为什么要这样做，以及我认为我们将在整个行业中看到更多这样的情况。

新闻

Fin 是一个客户 Agent 平台，目前每周解决超过 200 万个客户问题，并以快速指数级速度增长。它受到你能想象的每个垂直领域中最好的品牌（无论大小）的信赖。从 Atlassian 和 Riot Games，到较小的新锐公司如 Mercury 和 Polymarket。它运行在我们 AI 团队训练的模型系列上。上周，我们宣布了 Apex，这是世界上第一个专门的客户服务 LLM。在过去 6 个月的生产测试中，它在解决率、延迟、幻觉率和成本方面击败了每一个前沿模型，包括来自 Anthropic 和 OpenAI 的模型。

我们现在将允许您通过 API 直接访问所有这些功能和我们的所有核心模型，合同起价为每年 25 万美元，使用费率是每个模型子类别中业内最便宜的。

但为什么？

仅仅是因为我们的客户想要它。我们听到来自各地的人想要构建自己的 Agent。所以从今天开始，我们提供三种方法来实现这一目标。

首先，对于绝大多数公司来说，他们希望在 Fin Agent 平台上运行他们的运营。今天我们在这个平台上约有 8000 家公司。这满足了 99% 客户的需求，并允许他们轻松配置，而无需我们初创竞争对手那样令人筋疲力尽的咨询合作。它提供业内最好的解决率，而且是开箱即用的。

其次，我们也为那些想在独特环境中展示 Fin 的人提供了解决方案。为此，他们可以使用 Fin Agent API。在这种模式下，您可以获得 Fin 平台的所有魔力，但不必使用我们的 Messenger（或我们的电子邮件或语音或其他预建渠道），并可以以定制方式展示 Agent。

但也有一些公司希望为他们的业务构建超特定和专业的 Agent。也许他们想构建一个既能提供服务又是产品 Agent 的 Agent，让用户与他们的产品互动。在这种情况下，对他们来说最好和最明显的决定是使用 Apex 和我们在更广泛系统中使用的模型集合，因为它们正是为此目的训练的——不同于通用模型。这是我们今天推出的第三个新产品。

我们也很高兴看到新的初创公司构建类似 Fin 的业务，迎合超特定的垂直领域。牙医的 Fin？为什么不呢？汽车经销商的 Fin？当然可以。我们永远不会为这些特定垂直领域构建，但我们希望其他人这样做。事实上，如果我们任何直接的初创竞争对手想要大幅改进他们的产品并给我们一点分成，我们也非常乐意授权我们的模型给他们。Decagon、Sierra 和其他公司，你们知道在哪里找到我。让我们做朋友吧！:)

即将来到您附近的 Agent 公司；防御性原因

与许多人不同，我不是 100% 准备否定所有软件。但确实可以说，软件景观肯定将在我们眼前发生巨大变化。在最近的时间里，差异化来自软件功能，这些功能充当护城河，因为它们很难构建。但构建软件现在变得不那么难了。

我们已经成功地将工程团队的可衡量生产力提高了一倍以上。我们还创建了极其深入的新产品，这些产品以前作为独立业务存在，由单个 Intercom 工程师在短短一周内构建。

展望未来，来自功能和界面的差异化至少会减少。严肃的软件公司不仅要从功能公司转变为 Agent 公司，因为他们过去协助的人类工作现在将由 Agent 完成，而且还必须在这些 Agent 上构建差异化的 AI。我们确实相信越来越多的价值将累积到模型层，因此，正如我们开始用 Agent 业务颠覆我们的软件业务时那样，我们现在将开始用我们的 AI 业务颠覆我们的 Agent 业务的过程。

这一切会在哪里结束，任何人都可以猜测，但很难不想象我们也会在许多其他公司看到这种情况。目前，我们很高兴能率先推出这个新平台和业务，我们迫不及待地想看看人们会构建什么。
