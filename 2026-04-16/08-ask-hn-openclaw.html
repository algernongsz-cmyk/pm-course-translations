bryan0:

想想你希望助手做什么。你可以教它使用任何可用的 API 完成基本任务，然后你可以给它反馈让它改进。

例如，我的 agent 可以通过 Home Assistant 或任何其他 API 控制家庭自动化。我的 agent 为网站和开源项目做贡献。当你给它反馈时，它会更新自己的技能文件。

它会检查和回复邮件，可以接听和拨打电话，进行在线研究和监控。我甚至尝试让它创作音乐。可以尝试的事情是无限的。

我认为就像 LLM 一样，人们会在它不能一次性解决问题时感到沮丧。这项技术在反馈中成长。它会犯错，你的工作是确保它从这些错误中学习，不再重复。

lexandstuff:

我仍然在使用它，觉得它很有帮助。

我的 OpenClaw 实例使用一个 Obsidian 项目作为记忆。主要是，它只是我日常使用的 LLM，我通过 WhatsApp 访问它，但记忆没有被锁定在特定供应商那里，而是存储在版本控制中，我可以阅读和编辑。仅此原因就让它对我很有吸引力。当更好的 LLM 出现时，我只需要切换，我的记忆和系统提示词会随之迁移。

但是，我也用它来追踪卡路里/体重/锻炼、待办事项列表（账单、生日、活动提醒），以及支持各种生活管理任务。我几乎没有给它太多访问权限，除了几个只读访问某些数据的技能。

它没有给我带来 10 倍的生产力提升。它只是很方便。

dsiegel2275:

我已经在备用的一台 Macbook Pro 上安装了它。目前我只在一个用例中使用它：

每天晚上，我让 OpenClaw 从我的 Obsidian 笔记库的私有 GitHub repo 中拉取最新更改。然后它查看添加了哪些新笔记，然后运行"创建闪卡"技能来提取和编写有用的间隔重复练习闪卡。然后我给它访问我自己构建的基于 Web 的间隔重复闪卡应用程序的权限。它使用 API 插入新卡片，检查我上次复习卡片的时间，如果我几天没有复习卡片，可以选择发送提醒。

这是一个很好的工作流程，一直运行良好。我去上课，在 Obsidian 中做笔记，检查我的更改。到第二天，当我打开手机上的闪卡应用程序时，我有来自昨天课程的新卡片要复习。

mjsweet:

我是一名专业园艺师，我在 Mac 上使用 NanoClaw 做以下事情：

使用自定义 MCP 在我的工作管理工具（我自己做的）上安排工作。我会问它"今天有什么工作"等问题，开始工作，完成工作等。

它会通过 MCP 监控来自当地房地产经纪人工作订单的 Gmail，然后安排报价访问。

报价访问后，我将照片添加到 Telegram（我碰巧使用的频道），然后我让它分析照片。

NanoClaw 中的 Claude 在弄清楚需要做什么方面做得很好，但并不总是完全正确，所以我使用 intake-api 这是一个"session inbox"，它生成一个表单并将其与图像一起上传到 Cloudflare，并在聊天中放置表单链接，这样我可以对属性的注释图像进行调整。在我满意后，我点击提交。

我会回到 Telegram 频道，告诉它我已经提交了表单。Claude 会将 JSON 有效负载拉回到 session 中并将其集成到报价中。

它会问我工作成本等问题，以及它认为需要知道的任何其他事情。

然后它会使用 Latex 生成长度在 14 到 32 页之间的完整 PDF 提案，具体取决于拍摄了多少照片。

有条款和条件部分，以及一些销售废话等，还有一个相当不错的封面页，上面有他们的联系信息和我的信息等。

当我满意时，我会让它创建一封带有附加 PDF 提案的 Gmail 草稿。

我会快速审查它，然后发送。

我还设置了 Xero MCP，所以我可以让它创建发票和联系人等。

我在上下车时做所有这些事情。

它释放了我的家庭生活，让我有更多时间陪伴孩子和妻子。

对于需要这些类型代理助手任务的小企业主来说，有很多空间，通过我的 Agentic CRM，我认为我看到了像我这样的人的未来一瞥。

bitmasher9:

我使用它。我从不向任何人推荐它，但这是一个我能从中获得价值的项目。有一些对这个项目的非常好的批评，对我来说最切中要害的是消耗 tokens 的方面和调整的方面。

最常见的问题是"你用它做什么"，所以这里是我的答案。

1. 我有流行的自托管人群的 Obsidian/OpenClaw 设置。我在 OpenClaw 中有大量的"cron"任务来获取数据并将其插入 Obsidian，或者总结我完成的 Obsidian 项目，或者提醒我关于 Obsidian 中的待办事项。这是我从自动化方式在 .md 文件层交互中获得最大价值的地方。例如，我有一个 cron 会将我的每日笔记总结成每周笔记，我的每周笔记总结成每月笔记。

2. 邮件收件箱管理。我有从某些人或主题提醒我邮件的工作。根据模糊的 LLM 规则将邮件处理到文件夹的工作等。

我还有一个个人教练 agent，使用 gcal 追踪我的体重、每周锻炼，并创建餐饮计划，推送到 mealie，这样我就知道买什么杂货和做什么菜。

文献综述：我描述一个研究问题，它运行完整的管道——搜索 Semantic Scholar + Google Scholar，创建具有干净元数据的 Zotero 集合，然后通过 9 种不同策略尝试获取 PDF（机构仓库、arXiv、Unpaywall、使用我大学凭据的 EZproxy 等）。自动获得约 60-65% 的 PDF 覆盖率。

我有一个叫 Betty 的私人购物 agent，它的角色是出去找到我想买东西的优惠。

我还用它运行研究项目的数据管道。它被指示使用 openai/gpt-5.4 与 beads 和 gastown 进行编码。

我仍然需要弄清楚如何高效管理模型切换。我还没到那一步。

这是第一个真正改变我工作方式的 AI 设置，而不是仅仅成为一个花哨的搜索引擎。

总结

这篇 Hacker News 讨论展示了 OpenClaw 使用者的真实反馈。有些人找到了实用场景（如 Obsidian 集成、邮件管理、个人助理），而另一些人则因为稳定性、成本和隐私问题放弃使用。总体来看，OpenClaw 对某些用户有价值，但对技术背景的用户来说，可能更倾向于使用更简单的替代方案（如 Claude Code + cron jobs）。
