Libretto 是一个用于构建健壮 Web 集成的工具包。它为你的编码 agent 提供了一个实时浏览器和一个 token 高效的 CLI，可以：

  • 以最小的上下文开销检查实时页面
  • 捕获网络流量以逆向工程网站 API
  • 记录用户操作并将其重放为自动化脚本
  • 针对真实网站交互式调试损坏的工作流

我们在 Saffron Health 构建 Libretto，用于帮助我们维护与常见医疗软件的浏览器集成。我们将其开源，以便其他团队能更轻松地做同样的事情。

安装

npm install libretto

# 首次设置：安装 skill、下载 Chromium 并固定默认快照模型
npx libretto setup

# 随时检查工作区就绪状态
npx libretto status

# 手动更改快照分析模型（高级覆盖）
npx libretto ai configure <openai | anthropic | gemini | vertex>

setup 会检测可用的提供商凭证（如 OPENAI_API_KEY），并自动将默认模型固定到 .libretto/config.json。在健康工作区重新运行 setup 会显示当前配置，而不是重新提示。如果之前配置的提供商缺少凭证，setup 会提供交互式修复流程。

当你想显式切换提供商或设置自定义模型字符串时，使用 ai configure。

使用 Libretto skill。去 LinkedIn 抓取前 10 条帖子的内容、发布者、反应数、前 25 条评论和前 25 条转发。

你的编码 agent 会打开一个窗口让你登录 LinkedIn，然后自动开始探索。

Libretto 可以读取你在浏览器中执行的操作，因此你可以执行一个工作流，然后要求它使用你的操作重建该工作流。

我们在 ./integration.ts 有一个浏览器脚本，自动化访问 Hacker News 并获取前 10 条帖子。将其转换为直接的网络脚本。使用 Libretto skill。

Libretto 可以从浏览器读取网络请求，这些请求可用于逆向工程 API 并创建直接调用这些请求的脚本。直接进行 API 调用比 UI 自动化更快、更可靠。你还可以要求 Libretto 进行安全分析，分析请求中的常见安全 cookie，以便了解网络请求方法是否安全。

我们在 ./integration.ts 有一个浏览器脚本，应该去 Availity 并为患者执行资格检查。但运行时我收到了损坏的选择器错误。修复它。使用 Libretto skill。

Agent 可以使用 Libretto 重现故障、在任何时候暂停工作流、检查实时页面并修复问题，全部自主完成。

CLI 参考

你也可以直接从命令行使用 Libretto。所有命令都接受 --session <name> 来指定特定会话。

npx libretto setup          # 交互式首次运行设置；自己运行，不要通过 agent
npx libretto status          # 检查 AI 配置健康状态和打开的会话
npx libretto open <url>      # 启动浏览器并打开 URL（默认有头模式）
npx libretto snapshot --objective "..." --context "..."  # 捕获 PNG + HTML 并用 LLM 分析
npx libretto exec "<code>"   # 对打开的页面执行 Playwright TypeScript（单引号参数）
echo "<code>" | npx libretto exec -  # 从 stdin 读取 Playwright TypeScript
npx libretto run <file>      # 运行文件的默认导出工作流
npx libretto resume          # 恢复暂停的工作流
npx libretto pages           # 列出会话中打开的页面
npx libretto save <domain>   # 保存浏览器会话（cookies、localStorage）以供重用
npx libretto close           # 关闭浏览器
npx libretto ai configure <provider>  # 手动更改快照分析模型
npx libretto status          # 显示 AI 配置和打开的会话

配置

所有 Libretto 状态都存在于项目根目录的 .libretto/ 目录中。配置存储在 .libretto/config.json。

.libretto/config.json 控制快照分析和视口设置：

{
  "version": 1,
  "ai": {
    "model": "openai/gpt-5.4",
    "updatedAt": "2026-01-01T00:00:00.000Z"
  },
  "viewport": { "width": 1280, "height": 800 }
}

ai 字段配置 Libretto 用于快照分析的模型——提取选择器、识别交互元素或诊断步骤失败原因。这将重型视觉上下文保留在你的编码 agent 的上下文窗口之外。快照分析是必需的。

npx libretto setup 自动固定它找到凭证的第一个提供商的默认模型。之后要显式更改提供商或模型：

npx libretto ai configure <openai | anthropic | gemini | vertex>

要检查当前配置而不更改任何内容：

npx libretto status

提供商凭证从环境变量或仓库根目录（.git 目录旁边）的 .env 文件中读取：OPENAI_API_KEY、ANTHROPIC_API_KEY、GEMINI_API_KEY / GOOGLE_GENERATIVE_AI_API_KEY，或 Vertex 的 GOOGLE_CLOUD_PROJECT。设置 LIBRETTO_DISABLE_DOTENV=1 以跳过 .env 加载。

viewport 字段设置默认浏览器视口大小。两个字段都是可选的。

会话

每个 Libretto 会话都在 .libretto/sessions/<name>/ 下获得自己的目录，包含运行时状态。会话被 git 忽略。

  • state.json — 会话元数据（调试端口、PID、状态）
  • logs.jsonl — 结构化会话日志
  • network.jsonl — 捕获的网络请求
  • actions.jsonl — 记录的用户操作
  • snapshots/ — 截图 PNG 和 HTML 快照

配置文件

配置文件保存浏览器会话（cookies、localStorage），以便你可以在运行之间重用认证状态。它们存储在 .libretto/profiles/<domain>.json，通过 npx libretto save <domain> 创建。配置文件是机器本地的，并被 git 忽略。

开发

在此仓库中进行本地开发：

pnpm i
pnpm build
pnpm type-check
pnpm test

源码布局：

  • packages/libretto/src/cli/ — CLI 命令
  • packages/libretto/src/runtime/ — 浏览器运行时（网络、恢复、下载、提取）
  • packages/libretto/src/shared/ — 共享工具（配置、LLM 客户端、日志、状态）
  • packages/libretto/test/ — 测试文件（*.spec.ts）
  • packages/libretto/README.template.md — 仓库和包 README 的真实来源
  • packages/libretto/skills/libretto/ — Libretto skill 的真实来源

编辑 packages/libretto/README.template.md 或 packages/libretto/skills/libretto/ 下的任何内容后，运行 pnpm sync:mirrors。

要检查生成的 README、skill 镜像和 skill 版本元数据是否同步而不修复它们，运行 pnpm check:mirrors。要发布，运行 pnpm prepare-release。
